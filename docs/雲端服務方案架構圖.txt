# OmniBot AI 雲端服務方案架構圖

## 一、系統架構總覽

OmniBot AI 雲端架構包含以下主要層次：

1. CDN 層 - CloudFlare 靜態資源分發
2. 負載均衡層 - Nginx/ALB (Port 80/443)
3. 應用服務層
   - Admin Dashboard (React/Vite)
   - Chat Widget (React/Vite)
   - API Gateway (NestJS)
4. 業務邏輯層
   - API Service (NestJS)
   - AI Engine (FastAPI)
   - Message Queue (RabbitMQ)
   - Cache Layer (Redis)
5. 資料層
   - Vector Database (Qdrant)
   - Relational DB (PostgreSQL)

---

## 二、多租戶架構

### 2.1 資料庫隔離

PostgreSQL 採用 Schema-based 隔離：
- 每個租戶擁有獨立的 Schema
- Schema 內包含：bots, conv, messages, surveys, channels 等表格

Qdrant 採用 Namespace-based 隔離：
- 每個租戶的向量資料獨立儲存
- 透過 namespace 區分不同租戶的嵌入向量

---

## 三、訊息流程架構

訊息處理流程：

1. User Device 發送訊息
2. Channel Webhook 接收
3. API Service 處理
4. 訊息排入 Queue
5. AI Engine 生成回覆
6. 回傳 API Service
7. 儲存並回覆用戶

---

## 四、部署架構 (Kubernetes)

### 4.1 Kubernetes 元件

- Ingress Controller (Nginx/Traefik)
- API Deployment (3-5 Replicas)
- AI Engine Deployment (GPU Enabled)
- Admin Deployment (2 Replicas)
- Widget Deployment (2 Replicas)

### 4.2 資料層

- PostgreSQL Primary + Replica (3 Nodes)
- Redis Cluster (3 Masters)
- Qdrant Cluster (3 Nodes)
- Object Storage (S3/MinIO)
- CDN (CloudFlare)

---

## 五、網路架構

### 5.1 網路分區

1. 公網 (Public Subnet)
   - Web Browser
   - Mobile App
   - LINE Webhook
   - Facebook Webhook

2. 私有網路 (Private Subnet)
   - Kubernetes Cluster
   - Application Services
   - Data Layer

3. 管理網路 (Management Subnet)
   - Admin PC
   - DevOps Workstation
   - Monitoring Server

---

## 六、監控架構

### 6.1 監控指標收集

- Node Exporter - 節點監控
- Container Exporter - 容器監控
- Application Exporter - 應用監控
- Business Metrics - 業務指標

### 6.2 日誌收集

- 各服務日誌 → Fluentd/B → Elasticsearch → Kibana

### 6.3 警報通知

- Alertmanager 統一管理
- 支援 Email, Slack, PagerDuty, SMS

---

## 七、災難復原架構

### 7.1 部署策略

- Primary Region: AWS us-east-1
- DR Region: AWS us-west-2
- 異地備份 (RDS + Qdrant + S3)

### 7.2 復原目標

- RTO (還原時間目標): < 4 小時
- RPO (還原點目標): < 24 小時
- 自動故障轉移
- 定期演練

---

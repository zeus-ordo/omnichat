## 一、服務說明

OmniBot AI 是一家基於 AI 的多租戶客服機器人系統，提供以下核心服務：

### 1.1 服務項目

| 服務類別 | 服務內容 |
|----------|----------|
| AI 對話服務 | GPT-4o、Claude 3.5 Sonnet 模型對話 |
| 知識庫服務 | 文件上傳、RAG 問答、向量搜尋 |
| 多渠道整合 | LINE、Facebook Messenger、REST API、Web Widget |
| 管理後台 | 機器人設定、對話管理、數據分析 |
| 即時通訊 | WebSocket 雙向即時訊息 |

## 二、服務可用性

### 2.1 可用性承諾

| 服務層級 | 月可用性 | 停機時間 (每月) |
|----------|----------|-----------------|
| 企業版 | 99.9% | 43 分鐘 |
| 專業版 | 99.5% | 3.6 小時 |
| 基本版 | 99.0% | 7.3 小時 |

### 2.2 服務等級計算

可用性 = (總運行時間 - 總停機時間) / 總運行時間 × 100%

計算週期: 每月計算
排除項目:
- 計劃性維護 (提前 72 小時通知)
- 不可抗力因素 (天災、戰爭、重大公共衛生事件)
- 客戶端問題 (網路、设备故障)
- 第三方服務商問題 (OpenAI、Anthropic API 異常)

### 2.3 各服務可用性指標

| 服務元件 | 目標可用性 | 說明 |
|----------|------------|------|
| API 服務 | 99.9% | REST API 響應 |
| AI 引擎 | 99.5% | AI 模型回應 |
| 管理後台 | 99.9% | Web 介面訪問 |
| Web Widget | 99.9% | 嵌入聊天元件 |
| 資料庫 | 99.99% | 資料讀寫 |
| 向量資料庫 | 99.9% | 語義搜尋 |
| 快取服務 | 99.9% | 會話/狀態快取 |

## 四、客戶支援服務

### 4.1 支援等級

| 等級 | 適用方案 | 響應時間 | 支援時段 |
|------|----------|----------|----------|
| 企業版 | 企業方案 | < 1 小時 | 7×24 小時 |
| 專業版 | 專業方案 | < 4 小時 | 5×8 小時 |
| 基本版 | 基本方案 | < 24 小時 | 電子郵件 |

### 4.2 問題分類

| 級別 | 定義 | 響應時間 | 解決目標 |
|------|------|----------|----------|
| P0 - 緊急 | 系統完全不可用 | < 1 小時 | 4 小時內恢復 |
| P1 - 高 | 核心功能受影響 | < 2 小時 | 8 小時內解決 |
| P2 - 中 | 部分功能異常 | < 8 小時 | 24 小時內解決 |
| P3 - 低 | 輕微問題/建議 | < 24 小時 | 下一版本修復 |

### 4.3 支援渠道

| 渠道 | 可用性 |
|------|--------|
| 線上客服 | 企業版、專業版 |
| 電子郵件 support@omnibot.ai | 全方案 |
| 電話專線 | 企業版 |
| 技術文件 | https://docs.omnibot.ai |

## 五、資料保護與備份

### 5.1 資料備份策略

| 資料類型 | 備份頻率 | 保留期限 | 備份位置 |
|----------|----------|----------|----------|
| 資料庫 (PostgreSQL) | 每日 + 即時 WAL | 30 天 | 異地備份 |
| 向量資料庫 (Qdrant) | 每日 | 14 天 | 異地備份 |
| 快取 (Redis) | 無 (易失性) | - | - |
| 文件上傳 | 每日 | 90 天 | 物件儲存 |

### 5.2 資料還原目標

| 指標 | 目標值 |
|------|--------|
| 還原時間目標 (RTO) | < 4 小時 |
| 還原點目標 (RPO) | < 24 小時 |

### 5.3 資料安全

| 安全措施 | 說明 |
|----------|------|
| 傳輸加密 | TLS 1.3 |
| 儲存加密 | AES-256 |
| 資料隔離 | Schema 隔離 |
| 存取控制 | RBAC 權限管理 |
| 審計日誌 | 90 天保留 |

## 六、變更管理

### 6.1 計劃性維護

| 類型 | 通知時間 | 維護時段 |
|------|----------|----------|
| 例行維護 | 72 小時 | 凌晨 02:00-04:00 (UTC+8) |
| 緊急修補 | 12 小時 | 必要時 |
| 重大升級 | 7 天 | 協調客戶時間 |

### 6.2 服務中斷補償

| 可用性達標 | 補償方式 |
|------------|----------|
| 99.0% - 99.5% | 當月 10% 服務折扣 |
| 95.0% - 99.0% | 當月 25% 服務折扣 |
| < 95.0% | 當月 50% 服務折扣 |

補償上限: 單月不超過免費服務價值
申請期限: 故障發生後 30 天內
